
**Кэш (cache)** — это сохранённые результаты предыдущих операций, чтобы **не пересобирать или не скачивать одно и то же** при каждом деплое.


## ⚡ 3. Виды кэша

|Вид|Описание|Пример|
|---|---|---|
|**Локальный кэш**|Хранится на машине, где выполняется сборка|`~/.cache/pip`, `~/.gradle/caches`|
|**CI/CD кэш**|Хранится на сервере CI и подтягивается по ключу (hash)|GitLab `cache:key`, GitHub Actions `actions/cache`|
|**Docker-кэш**|Хранится в виде слоёв образа|`docker build` использует старые слои|
|**Remote cache (удалённый)**|Сохраняется на удалённом сервере (S3, Nexus, Artifactory)|Gradle remote build cache, npm registry|
|**Content cache (контентный)**|Основан на хэшах содержимого (если содержимое не менялось — не пересобираем)|Doc|
