
EXPLAIN ANALYZE -- Это команда в Postgres которая используеться для анализа и оптимизации выполнения SQL - Запросов. Она предоставляет информацию о том как база данных планирует и выполняет запрос включая оценку стоимости и времени выполнения.

КАК ПОНЯТЬ ЧТО ЗАПРОС ТУПИТ?
Я построю дерево испольнение запросов и посмотрю какие команды выполняються 
Я посмотрю на узлы которые выполняються в плане

1) ### **Seq Scan (Sequential Scan)**
Seq Scan on users  (cost=0.00..1000.00 rows=100000 width=...)
- Проблема: PostgreSQL **читает всю таблицу построчно**, потому что **нет нужного индекса**.

1) Отвечаю чётко: **`Index Scan`** появляется в `EXPLAIN` / `EXPLAIN ANALYZE`, **если PostgreSQL использует индекс**, чтобы быстро найти нужные строки **вместо того, чтобы читать всю таблицу**.

- PostgreSQL **идёт в индекс**, находит нужную строку, потом **идёт в таблицу** (heap), чтобы добрать остальные колонки.
- Это **гораздо быстрее**, чем `Seq Scan`, особенно на больших таблицах.

Основные компоненты вывода:

### 1. **Cost (стоимость):**

- **Начальная стоимость:** `cost=0.00..10.00` (первое значение).
- **Конечная стоимость:** `cost=0.00..10.00` (второе значение).
- Стоимость включает CPU, I/O операции и другие ресурсы.

### 2. **Rows (строки):**

- Ожидаемое количество строк.
- Полезно для оценки эффективности индексов.

### 3. **Width (ширина):**

- Средний размер строки в байтах.
- Указывает на объём данных, передаваемых между узлами.

### 4. **Actual Time (фактическое время):**

- Время выполнения узла, от начала до конца.
- Полезно для оценки реальной производительности.

### 5. **Loops (циклы):**

- Количество раз, которое узел был выполнен.
- Важно для понимания вложенных циклов и повторного использования.
