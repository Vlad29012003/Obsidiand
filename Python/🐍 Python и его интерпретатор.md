**Интерпретатор** — это программа, которая **сразу выполняет код построчно**, без предварительной генерации отдельного машинного кода. В Python это, например, `CPython`

Пример работы интерпретатора:

`x = 10 y = 5 print(x + y)`

- Интерпретатор читает первую строку `x = 10`, создаёт переменную `x` и присваивает ей значение.
    
- Потом строку `y = 5` — то же самое.
    
- Потом строку `print(x + y)` — вычисляет выражение `x + y` и выводит результат.
    

> То есть интерпретатор **не создаёт отдельный файл с машинным кодом**, он выполняет команды сразу.


### 2️⃣ Компиляция

**Компилятор** — это программа, которая **переводит весь исходный код в машинный код (или промежуточный код) заранее**, создавая исполняемый файл.


### 3️⃣ Отличия интерпретации и компиляции

| Характеристика   | Интерпретация (Python)                                    | Компиляция (C/C++)                               |
| ---------------- | --------------------------------------------------------- | ------------------------------------------------ |
| Время выполнения | Код выполняется построчно                                 | Исполняемый файл уже готов                       |
| Ошибки           | Выдаются при выполнении той строки, где ошибка            | Компилятор покажет ошибки до запуска             |
| Машинный код     | Не создаётся напрямую (может быть байткод `.pyc`)         | Создаётся исполняемый файл                       |
| Гибкость         | Очень гибкий, легко менять код на лету                    | Меньше гибкости, требуется пересборка            |
| Портативность    | Скрипт можно запускать на любой системе с интерпретатором | Файл зависит от ОС/архитектуры, нужен перекомпил |


### 4️⃣ Как Python всё же компилирует код

Немного хитро: **Python всё-таки делает промежуточную компиляцию**:

1. Скрипт `.py` → компилируется в **байткод** `.pyc` (интерпретатор CPython делает это автоматически).
    
2. Байткод выполняется **виртуальной машиной Python (PVM)**.
    
3. Благодаря этому байткод переносим между системами, но не является нативным машинным кодом.
    
То есть Python — это **интерпретируемый язык с компиляцией в байткод**.
